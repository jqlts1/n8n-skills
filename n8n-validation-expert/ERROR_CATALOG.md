# 错误目录

这份目录只保留新版 MCP 下最常见、最值得优先判断的错误类型。

---

## 1. 代码语法错误

### 典型表现

- 无法解析工作流代码
- 第一条错误就卡死后续验证

### 常见原因

- 少逗号
- 少括号
- 引号不闭合
- 对象或数组层级写乱

### 处理顺序

先把语法修到验证器能完整读代码，再看后面的业务错误。

---

## 2. 节点类型未找到

### 典型表现

- 节点 ID 不存在
- 节点名称像是猜的

### 常见原因

- 把旧节点名直接搬过来
- 记错官方节点 ID

### 修法

```text
search_nodes -> get_node_types -> 用确认过的节点 ID 重写
```

---

## 3. 参数名或参数类型不匹配

### 典型表现

- 字段名不存在
- 某字段需要对象，却给了字符串
- 某字段在当前操作下不生效

### 常见原因

- 没有针对具体操作读取类型定义
- 把旧版 JSON 结构直接移植过来

### 修法

```javascript
// 先看当前操作的类型，再回填 node(...)
node('n8n-nodes-base.httpRequest', {
  method: 'POST',
  url: 'https://api.example.com',
  sendBody: true,
  body: {
    contentType: 'json',
    content: {
      name: '={{$json.name}}',
    },
  },
});
```

---

## 4. 缺少必填字段

### 典型表现

- 某个操作看起来差不多，但就是过不了验证

### 常见原因

- 换了 `resource` / `operation`
- 认证方式变了
- 打开了某个开关，却没补后续字段

### 修法

先缩回最小可运行形状，再逐步补字段。

---

## 5. 连接错误

### 典型表现

- 节点没有接上主链路
- 分支接反
- AI 节点特殊连接类型错了

### 修法

- 先保证主执行路径连通
- 特殊节点再看专门的连接规则

特别是：

- Loop Over Items
- Merge
- AI Agent / Model / Parser / Tool

---

## 6. 表达式错误

### 典型表现

- 表达式被当成普通文本
- 运行时数据取不到
- 校验提示引用无效

### 常见原因

- 漏写 `{{ }}`
- 数据路径写错
- 误把别的节点名当当前输入

### 修法

先确认当前节点实际输入，再写表达式，不要靠猜。

---

## 7. 警告类问题

### 常见类型

- 结构虽然合法，但不稳
- 某个参数设置容易引起运行时问题
- 缺少明显的错误处理

### 处理建议

如果当前目标是先创建成功，可以先修 errors，再回头清 warnings。

---

## 快速判断表

| 现象 | 先查什么 |
|------|----------|
| 一上来就很多错 | 先查语法 |
| 节点名字眼熟但过不了 | 先查节点 ID |
| 字段好像对但总报错 | 先查当前操作的类型定义 |
| 分支逻辑不对 | 先查连接 |
| 值写出来却没生效 | 先查表达式 |
